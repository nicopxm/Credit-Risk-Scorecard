# Credit Risk Scorecard — Probability of Default Model

> A data-driven credit scoring system built in Python to score loan applicants by probability of default, reduce portfolio default rates, and provide a standardised alternative to manual loan review processes.

---

## Project Overview

A digital lender's 8.1% default rate across 307,511 applications was exceeding risk thresholds and eroding profitability. At a 10% margin on performing loans, every default requires ten successful loans to break even, meaning the business needed to originate volume equivalent to 80% of its current portfolio just to recover existing losses.

This project delivers a **Probability of Default (PD) scorecard** built using logistic regression and Weight of Evidence (WoE) encoding, developed across three iterative model phases. The final model reduces the portfolio default rate from 8.1% to **3.37%** on approved applicants while maintaining a **55.2% approval rate** — a 58% reduction in credit losses.

## Methodology

### Data
- **Source:** Home Credit Default Risk dataset (Kaggle)
- **Size:** 307,511 loan applications · 122 raw features
- **Target:** Binary default indicator (8.1% positive rate)

### Feature Engineering
All features WoE-encoded before model input. Final feature set of 14 across three phases:

| Phase | Features Added | IV Range |
|---|---|---|
| Phase 1 | Age, employment tenure, payment rate, education, gender, family status, income type, employed-to-age ratio | 0.022 — 0.097 |
| Phase 2 | EXT_SOURCE_1, EXT_SOURCE_2, EXT_SOURCE_3 | 0.136 — 0.314 |
| Phase 3 | EXT_SOURCES_MEAN, DAYS_LAST_PHONE_CHANGE, REGION_RATING_CLIENT | 0.028 — 0.059 |

### Model
- **Algorithm:** Logistic Regression (`class_weight='balanced'`)
- **Encoding:** Weight of Evidence (WoE) — no StandardScaler applied
- **Split:** Stratified 80/20 train/test
- **Selection rationale:** Logistic regression selected for full interpretability, WoE linearisation of log-odds relationship, and regulatory compliance for adverse action notices

### Threshold Optimisation
Decision threshold of **0.442** selected to simultaneously satisfy:
- Portfolio default rate ≤ 7.5%
- Approval rate ≥ 55%

## Fairness Note

`CODE_GENDER` is included in the model with an IV of 0.0388. Male applicants default at 10.1% vs 7.0% for females — a 44% higher relative rate. This feature is flagged for compliance review before any production deployment. Full fairness documentation is included in the model card.
